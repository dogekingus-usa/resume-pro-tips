## ATS-Friendly Formatting Rules

### 1. Use a Simple Single-Column Layout
Two-column layouts, sidebars, and complex tables confuse ATS parsers. The AI reads left-to-right, top-to-bottom. When information is split across columns, the parser jumbles it together, resulting in an unreadable mess. Stick to a clean, single-column layout with clear section breaks.



### 2. Standard Section Headings Only
ATS systems are trained to recognize standard section headings. Use exactly these:



- **Professional Summary** or **Objective**
- **Experience** or **Work History**
- **Education**
- **Skills**
- **Certifications** (if applicable)

Avoid creative headings like "What I Bring to the Table" or "My Journey." These break the parser's ability to categorize your information correctly.



### 3. Choose the Right Font and Size
ATS systems prefer standard fonts that all systems can render. Use:



- **Font:** Arial, Calibri, Helvetica, or Georgia
- **Size:** 10-12pt for body text, 14-16pt for headings
- **Formatting:** Bold and italic are fine, but avoid underlining (it can confuse parsers)
- **Avoid:** Text boxes, shapes, images, embedded charts, and word art


### 4. Save as DOCX (Not PDF for ATS Screening)
While PDFs preserve your formatting visually, many ATS systems struggle to extract text from PDFs reliably. For the best ATS compatibility, submit your resume as a **.docx file**. If the application system converts it, the DOCX will render correctly. Keep a PDF version ready for when a human specifically requests one.



### 5. Avoid Tables and Columns
Even simple tables can break ATS parsing. Instead of using a table to organize skills or dates, write them inline. For example, instead of a table with "Company | Role | Dates," use separate lines or bullet points.





**Quick Fix:** If you're using a template from Canva, Google Docs, or Microsoft Word, strip out any tables, text boxes, or columns before submitting. When in doubt, paste the content into a plain text editor to see what the ATS will actually read.

- **Include both acronyms and full names** â€” Write "Search Engine Optimization (SEO)" to catch both versions.

- Single-column layout â€” no columns or sidebars

- **Using images or icons** â€” Profile photos, logos, and icon sets are invisible to ATS and increase file size.

- **Text in headers/footers** â€” Many ATS systems skip header and footer content entirely.
- **Non-standard date formats** â€” Use "May 2026" or "05/2026" consistently. Never use relative dates like "Present."
